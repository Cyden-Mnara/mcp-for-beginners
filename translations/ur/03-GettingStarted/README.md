## شروعات  

[![اپنا پہلا MCP سرور بنائیں](../../../translated_images/ur/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(اس سبق کی ویڈیو دیکھنے کے لیے اوپر تصویر پر کلک کریں)_

اس حصے میں کئی اسباق شامل ہیں:

- **1 آپ کا پہلا سرور**، اس پہلے سبق میں، آپ سیکھیں گے کہ اپنا پہلا سرور کیسے بنائیں اور انسکیپٹر ٹول کے ذریعے اسے کیسے جانچیں، جو آپ کے سرور کی جانچ اور ڈیبگ کے لیے ایک قیمتی طریقہ ہے، [سبق پر جائیں](01-first-server/README.md)

- **2 کلائنٹ**، اس سبق میں، آپ سیکھیں گے کہ ایک کلائنٹ کیسے لکھا جائے جو آپ کے سرور سے جڑ سکے، [سبق پر جائیں](02-client/README.md)

- **3 LLM کے ساتھ کلائنٹ**، کلائنٹ لکھنے کا ایک بہتر طریقہ یہ ہے کہ اس میں LLM شامل کیا جائے تاکہ یہ "مذاکرہ" کر سکے کہ سرور کے ساتھ کیا کرنا ہے، [سبق پر جائیں](03-llm-client/README.md)

- **4 Visual Studio Code میں GitHub Copilot ایجنٹ موڈ کے ساتھ سرور استعمال کرنا**۔ یہاں ہم MCP سرور کو Visual Studio Code کے اندر چلانے پر غور کر رہے ہیں، [سبق پر جائیں](04-vscode/README.md)

- **5 stdio ٹرانسپورٹ سرور** stdio ٹرانسپورٹ مقامی MCP سرور سے کلائنٹ رابطے کے لیے تجویز کردہ معیار ہے، جو محفوظ subprocess پر مبنی رابطہ فراہم کرتا ہے اور عمل کے الگ تھلگ کرنے کی سہولت دیتا ہے [سبق پر جائیں](05-stdio-server/README.md)

- **6 MCP کے ساتھ HTTP سٹریمنگ (Streamable HTTP)**۔ جدید HTTP سٹریمنگ ٹرانسپورٹ کے بارے میں جانیں (جو [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) کے مطابق ریموٹ MCP سرورز کے لیے تجویز کردہ طریقہ ہے)، پیش رفت کی اطلاعیں، اور اسٹریمنگ ایبل HTTP کا استعمال کرتے ہوئے وسیع تر، ریئل ٹائم MCP سرورز اور کلائنٹس کیسے بنائیں۔ [سبق پر جائیں](06-http-streaming/README.md)

- **7 VSCode کے لیے AI Toolkit استعمال کرنا** تاکہ اپنے MCP کلائنٹس اور سرورز کو استعمال اور جانچ سکیں [سبق پر جائیں](07-aitk/README.md)

- **8 جانچ**۔ یہاں ہم خاص طور پر توجہ دیں گے کہ اپنے سرور اور کلائنٹ کی مختلف طریقوں سے جانچ کیسے کریں، [سبق پر جائیں](08-testing/README.md)

- **9 تعیناتی**۔ یہ باب آپ کے MCP حل کی تعیناتی کے مختلف طریقوں پر نظر ڈالے گا، [سبق پر جائیں](09-deployment/README.md)

- **10 جدید سرور کا استعمال**۔ یہ باب جدید سرور کے استعمال کا احاطہ کرتا ہے، [سبق پر جائیں](./10-advanced/README.md)

- **11 توثیق**۔ یہ باب سادہ توثیق شامل کرنے کا طریقہ بتاتا ہے، بنیادی توثیق سے JWT اور RBAC استعمال کرنے تک۔ آپ کو مشورہ دیا جاتا ہے کہ یہاں سے شروع کریں اور پھر باب 5 میں جدید موضوعات دیکھیں اور باب 2 میں سفارشات کے ذریعے اضافی سیکیورٹی سختی کریں، [سبق پر جائیں](./11-simple-auth/README.md)

- **12 MCP ہوسٹس**۔ معروف MCP ہوسٹ کلائنٹس جیسے Claude Desktop، Cursor، Cline، اور Windsurf کو ترتیب دیں اور استعمال کریں۔ ٹرانسپورٹ اقسام اور مسئلہ حل کرنا جانیں، [سبق پر جائیں](./12-mcp-hosts/README.md)

- **13 MCP انسکیپٹر**۔ MCP انسکیپٹر ٹول استعمال کرتے ہوئے اپنے MCP سرورز کو انٹرایکٹو طور پر ڈیبگ اور جانچیں۔ ٹولز، وسائل، اور پروٹوکول پیغامات کا مسئلہ حل کرنا سیکھیں، [سبق پر جائیں](./13-mcp-inspector/README.md)

- **14 سیمپلنگ**۔ MCP سرورز بنائیں جو MCP کلائنٹس کے ساتھ LLM سے متعلق کاموں پر تعاون کریں (جو `2026-07-28` ریلیز امیدوار میں ختم ہو رہا ہے؛ پھر بھی `2025-11-25` کے لیے جائز ہے)۔ [سبق پر جائیں](./14-sampling/README.md)

- **15 MCP ایپس**۔ MCP سرورز بنائیں جو یوزر انٹرفیس ہدایات کے ساتھ بھی جواب دیں، [سبق پر جائیں](./15-mcp-apps/README.md)

ماڈل کانٹیکسٹ پروٹوکول (MCP) ایک کھلا پروٹوکول ہے جو LLMs کو کانٹیکسٹ فراہم کرنے کے لیے معیاری طریقہ کار فراہم کرتا ہے۔ MCP کو AI کی ایپلی کیشنز کے لیے USB-C پورٹ سمجھیں - یہ AI ماڈلز کو مختلف ڈیٹا ذرائع اور آلات سے جوڑنے کا معیاری طریقہ فراہم کرتا ہے۔

## تعلیمی مقاصد

اس سبق کے اختتام تک، آپ کر سکیں گے:

- C#, Java, Python, TypeScript، اور JavaScript میں MCP کے لیے ترقیاتی ماحول تشکیل دیں
- بنیادی MCP سرورز کو حسب منشاء خصوصیات (وسائل، پرامپٹ، اور آلات) کے ساتھ بنائیں اور تعینات کریں
- میزبان ایپلیکیشنز بنائیں جو MCP سرورز سے جڑتی ہیں
- MCP کی تعمیل کی جانچ اور ڈیبگ کریں
- عام سیٹ اپ چیلنجز اور ان کے حل سمجھیں
- اپنے MCP نفاذات کو مشہور LLM خدمات سے مربوط کریں

## اپنا MCP ماحول تیار کرنا

MCP کے ساتھ کام شروع کرنے سے پہلے، اپنے ترقیاتی ماحول کو تیار کرنا اور بنیادی کام کا بہاؤ سمجھنا ضروری ہے۔ یہ حصہ آپ کو ابتدائی سیٹ اپ کے اقدامات کے ذریعے رہنمائی کرے گا تاکہ MCP کے ساتھ آسان آغاز یقینی بنایا جا سکے۔

### درکار تقاضے

MCP کی ترقی میں غوطہ لگانے سے پہلے یقینی بنائیں کہ آپ کے پاس یہ ہیں:

- **ترقیاتی ماحول**: اپنی منتخب زبان کے لیے (C#, Java, Python, TypeScript، یا JavaScript)
- **IDE/ایڈیٹر**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm، یا کوئی جدید کوڈ ایڈیٹر
- **پیکیج مینیجرز**: NuGet, Maven/Gradle, pip، یا npm/yarn
- **API کیز**: کسی بھی AI خدمات کے لیے جو آپ اپنے میزبان ایپلی کیشنز میں استعمال کرنا چاہتے ہیں


### سرکاری SDKs

آنے والے ابواب میں آپ Python, TypeScript, Java اور .NET استعمال کرتے ہوئے بنائے گئے حل دیکھیں گے۔ یہ ہیں تمام سرکاری طور پر سپورٹ شدہ SDKs۔

MCP مختلف زبانوں کے لیے سرکاری SDKs فراہم کرتا ہے (جو [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) کے مطابق ہیں):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - مائیکروسافٹ کے ساتھ تعاون میں برقرار رکھا جاتا ہے
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI کے ساتھ تعاون میں برقرار رکھا جاتا ہے
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - سرکاری TypeScript نفاذ
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - سرکاری Python نفاذ (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - سرکاری Kotlin نفاذ
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI کے ساتھ تعاون میں برقرار رکھا جاتا ہے
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - سرکاری Rust نفاذ
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - سرکاری Go نفاذ

## اہم نکات

- MCP ترقیاتی ماحول زبان کے مخصوص SDKs کے ساتھ ترتیب دینا آسان ہے
- MCP سرورز بنانا واضح اسکیماؤں کے ساتھ آلات تخلیق اور رجسٹر کرنے میں شامل ہے
- MCP کلائنٹس سرورز اور ماڈلز سے جڑتے ہیں تاکہ اضافی خصوصیات حاصل کریں
- جانچ اور ڈیبگ کرنا قابل اعتماد MCP تعمیل کے لیے ضروری ہے
- تعیناتی کے اختیارات مقامی ترقی سے لے کر کلاؤڈ بیسڈ حل تک پھیلے ہوئے ہیں
