این بخش شامل چندین درس است:

- **1 سرور اول شما**، در این اولین درس، یاد می‌گیرید چگونه اولین سرور خود را ایجاد کرده و با ابزار بازرس آن را بررسی کنید؛ روشی ارزشمند برای تست و اشکال‌زدایی سرور شما، [به درس](01-first-server/README.md)

- **2 کلاینت**، در این درس آموزش می‌بینید چگونه یک کلاینت بنویسید که بتواند به سرور شما متصل شود، [به درس](02-client/README.md)

- **3 کلاینت با LLM**، راهی بهتر برای نوشتن کلاینت این است که یک LLM به آن اضافه کنید تا بتواند با سرور شما «مذاکره» کند که چه کاری انجام دهد، [به درس](03-llm-client/README.md)

- **4 مصرف حالت Agent گیت‌هاب کوپایلوت برای سرور MCP در Visual Studio Code**. در اینجا، به اجرای سرور MCP خود از داخل Visual Studio Code می‌پردازیم، [به درس](04-vscode/README.md)

- **5 سرور انتقال stdio** انتقال stdio استاندارد پیشنهادی برای ارتباط محلی سرور MCP با کلاینت است که ارتباط امن مبتنی بر زیرفرآیند با جداسازی فرآیند داخلی فراهم می‌کند [به درس](05-stdio-server/README.md)

- **6 پخش HTTP با MCP (پخش‌پذیر HTTP)**. با انتقال پخش HTTP مدرن آشنا شوید (رویکرد پیشنهادی برای سرورهای دور MCP بر اساس [مشخصات MCP 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http))، اعلان‌های پیشرفت، و نحوه پیاده‌سازی سرورها و کلاینت‌های MCP مقیاس‌پذیر و زمان واقعی با استفاده از HTTP پخش‌پذیر. [به درس](06-http-streaming/README.md)

- **7 استفاده از کیت ابزار هوش مصنوعی برای VSCode** جهت مصرف و تست کلاینت‌ها و سرورهای MCP شما [به درس](07-aitk/README.md)

- **8 تست**. در اینجا به ویژه تمرکز می‌کنیم چگونه می‌توانیم سرور و کلاینت خود را به روش‌های مختلف تست کنیم، [به درس](08-testing/README.md)

- **9 استقرار**. این فصل به راه‌های مختلف استقرار راه‌حل‌های MCP شما می‌پردازد، [به درس](09-deployment/README.md)

- **10 استفاده پیشرفته از سرور**. این فصل به استفاده پیشرفته از سرور می‌پردازد، [به درس](./10-advanced/README.md)

- **11 احراز هویت**. این فصل به نحوه افزودن احراز هویت ساده، از Basic Auth تا استفاده از JWT و RBAC می‌پردازد. پیشنهاد می‌شود با اینجا شروع کنید و سپس به مباحث پیشرفته در فصل 5 نگاه کرده و سخت‌سازی امنیتی اضافی را از طریق توصیه‌های فصل 2 انجام دهید، [به درس](./11-simple-auth/README.md)

- **12 میزبان‌های MCP**. پیکربندی و استفاده از کلاینت‌های میزبان محبوب MCP از جمله Claude Desktop, Cursor, Cline و Windsurf. یادگیری انواع انتقال و عیب‌یابی، [به درس](./12-mcp-hosts/README.md)

- **13 بازرس MCP**. اشکال‌زدایی و تست سرورهای MCP خود را به صورت تعاملی با ابزار بازرس MCP انجام دهید. یاد بگیرید چگونه ابزارها، منابع و پیام‌های پروتکل را عیب‌یابی کنید، [به درس](./13-mcp-inspector/README.md)

- **14 نمونه‌برداری**. ایجاد سرورهای MCP که با کلاینت‌های MCP روی وظایف مرتبط با LLM همکاری می‌کنند (منسوخ شده در نسخه کاندید  `2026-07-28`; هنوز معتبر برای `2025-11-25`). [به درس](./14-sampling/README.md)

- **15 اپلیکیشن‌های MCP**. ایجاد سرورهای MCP که همچنین با دستورالعمل‌های رابط کاربری پاسخ می‌دهند، [به درس](./15-mcp-apps/README.md)

پروتکل زمینه مدل (MCP) یک پروتکل باز است که استانداردسازی می‌کند چگونه برنامه‌ها به LLMها زمینه می‌دهند. MCP را مانند یک پورت USB-C برای برنامه‌های هوش مصنوعی در نظر بگیرید - روشی استاندارد جهت اتصال مدل‌های هوش مصنوعی به منابع داده و ابزارهای مختلف فراهم می‌کند.

## اهداف یادگیری

در پایان این درس شما قادر خواهید بود:

- راه‌اندازی محیط‌های توسعه برای MCP در زبان‌های C#, Java, Python, TypeScript و JavaScript
- ساخت و استقرار سرورهای پایه MCP با ویژگی‌های سفارشی (منابع، دستورات، و ابزارها)
- ایجاد برنامه‌های میزبان که به سرورهای MCP متصل می‌شوند
- تست و اشکال‌زدایی پیاده‌سازی‌های MCP
- درک چالش‌های رایج راه‌اندازی و راه‌حل‌های آنها
- اتصال پیاده‌سازی‌های MCP خود به خدمات محبوب LLM

## راه‌اندازی محیط MCP شما

پیش از شروع کار با MCP، مهم است که محیط توسعه خود را آماده کنید و روند کاری پایه را درک کنید. این بخش شما را در مراحل اولیه راه‌اندازی راهنمایی می‌کند تا شروعی روان با MCP داشته باشید.

### پیش‌نیازها

قبل از شروع توسعه MCP، اطمینان حاصل کنید که موارد زیر را دارید:

- **محیط توسعه**: برای زبان انتخابی شما (C#, Java, Python, TypeScript، یا JavaScript)
- **IDE/ویرایشگر**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm یا هر ویرایشگر کد مدرن دیگر
- **مدیر بسته‌ها**: NuGet, Maven/Gradle, pip یا npm/yarn
- **کلیدهای API**: برای هر سرویس هوش مصنوعی که قصد استفاده در برنامه‌های میزبان خود دارید


### SDKهای رسمی

در فصل‌های آینده نمونه‌هایی خواهید دید که با استفاده از Python، TypeScript، Java و .NET ساخته شده‌اند. در اینجا تمامی SDKهای رسمی پشتیبانی شده آمده است.

MCP SDKهای رسمی را برای چندین زبان ارائه می‌دهد (مطابق با [مشخصات MCP 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/)):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - همراه با همکاری مایکروسافت نگهداری می‌شود
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - همراه با همکاری Spring AI نگهداری می‌شود
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - پیاده‌سازی رسمی TypeScript
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - پیاده‌سازی رسمی Python (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - پیاده‌سازی رسمی Kotlin
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - همراه با همکاری Loopwork AI نگهداری می‌شود
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - پیاده‌سازی رسمی Rust
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - پیاده‌سازی رسمی Go

## نکات کلیدی

- راه‌اندازی محیط توسعه MCP با SDKهای مخصوص زبان‌ها آسان است
- ساخت سرورهای MCP شامل ایجاد و ثبت ابزارها با ساختارهای واضح است
- کلاینت‌های MCP به سرورها و مدل‌ها متصل می‌شوند تا قابلیت‌های توسعه‌یافته را بهره‌برداری کنند
- تست و اشکال‌زدایی برای پیاده‌سازی قابل اعتماد MCP ضروری است
- گزینه‌های استقرار از توسعه محلی تا راه‌حل‌های مبتنی بر ابر متغیر است
